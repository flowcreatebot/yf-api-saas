Y Finance API — Customer Docs

For Zapier, Make, n8n, and custom HTTP modules.

Base URL

Use your deployed API domain (example): https://api.yfinanceapi.com

Authentication

Send your API key in the header:

x-api-key: YOUR_API_KEY

Endpoints

GET /v1/health

Health check endpoint.

GET /v1/quote/{symbol}

Current quote snapshot for one ticker.

curl -H "x-api-key: $API_KEY" https://api.yfinanceapi.com/v1/quote/AAPL

GET /v1/history/{symbol}

Historical OHLCV candles.

Query params: period (default 1mo), interval (default 1d), optional start/end (YYYY-MM-DD).

curl -H "x-api-key: $API_KEY" "https://api.yfinanceapi.com/v1/history/TSLA?period=3mo&interval=1d"

GET /v1/quotes?symbols=AAPL,MSFT,TSLA

Batch quote fetch. Maximum 25 symbols per request.

curl -H "x-api-key: $API_KEY" "https://api.yfinanceapi.com/v1/quotes?symbols=AAPL,MSFT,TSLA"

GET /v1/fundamentals/{symbol}

Company metadata and key valuation fields.

curl -H "x-api-key: $API_KEY" https://api.yfinanceapi.com/v1/fundamentals/MSFT

Billing Endpoints

GET /v1/billing/plans

Returns available plans.

POST /v1/billing/webhook/stripe

Stripe webhook receiver (server-to-server).

No-Code Setup Pattern

  1. Create HTTP request module in Zapier/Make/n8n.
  2. Add x-api-key header.
  3. Call quote/history/fundamentals endpoint.
  4. Map JSON fields into downstream actions.
